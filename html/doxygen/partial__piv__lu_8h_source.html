21 // Eigen-3.5+ provides better GPU support. However, some functions cannot be called directly
22 // from within an OpenACC region. Therefore, we need to wrap them in a special API (decorate
23 // them with __device__ & acc routine tokens), which allows us to eventually call them from OpenACC.
24 // Calling these functions from CUDA kernels presents no issue ...
25 
26 // We want to declare a function template that is callable from OpenMP and
27 // OpenACC code, but whose instantiations are compiled by CUDA. This is to avoid
28 // Eigen internals having to be digested by OpenACC/OpenMP compilers. The
29 // problem is that it is apparently not sufficient to declare a template in a OpenMP
30 // declare target region and have the OpenMP compiler assume that device
31 // versions of instantations of it will be available. The convoluted approach
32 // here has two ingredients:
33 // - partialPivLu<N>(...), a function template that has explicit
34 // instantiations visible in this header file that call:
35 // - partialPivLuN(...), functions that are declared in this header but defined
36 // in the CUDA file partial_piv_lu.cu.
37 nrn_pragma_omp(declare target)
38 nrn_pragma_acc(routine seq)
