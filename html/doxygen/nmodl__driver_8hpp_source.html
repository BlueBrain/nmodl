37 /**
38  * \class NmodlDriver
39  * \brief Class that binds all pieces together for parsing nmodl file
40  *
41  * Driver class bind components required for lexing, parsing and ast
42  * generation from nmodl file. We create an instance of lexer, parser
43  * and provides different methods to parse from file, stream or string.
44  * The scanner also gets reference to driver object for two purposes :
45  * scanner store/query the macro definitions into/from driver class
46  * and erros can be propogated back to driver (not implemented yet).
47  * Parser class also gets a reference to driver class as a parameter.
48  * Parsing actions generate ast and it's pointer is stored in driver
49  * class.
50  *
51  * \todo Lexer, parser and ast member variables are used inside lexer/
52  * parser instances. The local instaces are created inside parse_stream
53  * and hence the pointers are no longer valid except ast. Need better
54  * way to handle this.
55  *
56  * \todo Stream name is not used as it will need better support as
57  * location object used in scanner takes string pointer which could
58  * be invalid when we copy location object.
59  */
