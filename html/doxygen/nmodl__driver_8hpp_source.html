39 /**
40  * \class NmodlDriver
41  * \brief Class that binds all pieces together for parsing nmodl file
42  *
43  * Driver class bind components required for lexing, parsing and ast
44  * generation from nmodl file. We create an instance of lexer, parser
45  * and provides different methods to parse from file, stream or string.
46  * The scanner also gets reference to driver object for two purposes :
47  * scanner store/query the macro definitions into/from driver class
48  * and erros can be propogated back to driver (not implemented yet).
49  * Parser class also gets a reference to driver class as a parameter.
50  * Parsing actions generate ast and it's pointer is stored in driver
51  * class.
52  *
53  * \todo Lexer, parser and ast member variables are used inside lexer/
54  * parser instances. The local instaces are created inside parse_stream
55  * and hence the pointers are no longer valid except ast. Need better
56  * way to handle this.
57  *
58  * \todo Stream name is not used as it will need better support as
59  * location object used in scanner takes string pointer which could
60  * be invalid when we copy location object.
61  */
