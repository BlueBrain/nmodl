◆ DefinitionType

enum nmodl::details::DefinitionType
strong

Definition type similar to old implementation.

In the original implementation of NMODL (mod2c, nocmodl) different vectors were created for extdef, extdef2, extdef3, extdef4 etc. We are changing those vectors with <name, type> map. This will help us to search in single map and find it's type. The types are defined as follows:

  • DefinitionType::EXT_DOUBLE : external names that can be used as doubles without giving an error message
  • DefinitionType::EXT_2 : external function names that can be used with array and function name arguments
  • DefinitionType::EXT_3 : function names that get two reset arguments
  • DefinitionType::EXT_4 : functions that need a first arg of NrnThread*
  • DefinitionType::EXT_5 : external definition names that are not threadsafe
